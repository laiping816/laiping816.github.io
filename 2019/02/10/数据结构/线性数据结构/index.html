线性数据结构

发表于 2019-02-10 | 更新于: 2019-02-24 | 分类于 数据结构 | 浏览 次

时间复杂度、数组、栈、队列、链表

复杂度介绍

  • 衡量代码好坏指标

    • 运行时间（时间复杂度）
    • 占用空间（空间复杂度）

  • 时间复杂度

    由于运行环境和输入规模的影响，代码绝对时间无法估计，但可预估代码的基本操作执行次数

    计算得出T(n) = ….表达式

    • 渐进时间复杂度

      当n趋于 ∞ 的时候得到的极值

      • 运行时间为常数量级，O （1）
      • 只保留时间函数中最高阶项，且省去最高阶前系数
      • 常见有O（1）< O（logn）< O（n）< O（n^2）

线性数据结构

  • 底层依托静态数组

    （ArrayList、数组栈、数组队列）

    • 优
      • 有随机访问能力，index
    • 缺
      • 靠resize解决固定容量问题

  • 真正动态数据结构（无需考虑容量）

    （链表LinkedList、链表栈、链表队列）

    • 优
      • 真正动态，不需要处理固定容量问题
    • 缺
      • 丧失随机访问能力

动态数组

ArrayList

注意

  • 参数有index，都需要界限判断
  • 容量capacity不够，加倍; 删除导致容量过多，减半

  • 复杂度分析

    • 添加操作：O(n)

    • 删除操作：O(n)

      同添加

    • 修改操作：已知索引O(1)；未知索引O(n)

    • 查找操作：已知索引O(1)；未知索引O(n)

  • 均摊时间复杂度

    由于resize操作的存在，不可能每次添加都会触发。

    假设capacity = n，n+1次addLast，触发1次resize，总共进行2n+1次基本操作。

    （2n+1）/（n+1）≈ 2， 所以均摊时间复杂度为O(1)

  • 复杂度震荡的问题

    在capacity临界的地方，交替执行addLast和removeLast会出现频繁扩容缩容。

    解决：Lazy，当size == capacity / 4时，再减去一半。

LinkedList

  • 节点

★设立虚拟头结点dummyHead

  1. 为了解决，由于头位置没有前节点，有些操作（寻找Predecessor）需要区别对待的问题
  2. 为统一操作，故而添加

  • 时间复杂度

    • 增 O(n)
      • addLast O(n)
      • addFirst O(1)
      • add O(n/2)=O(n)
    • 删 O(n)
      • removeLast O(n)
      • removeFirst O(1)
      • remove O(n/2)=O(n)

    • 改 O(n)

    • 查 O(n)

      • get O(n)
      • contains O(n)

    小结

    1. 如果只对链表头进行操作：O(1)
    2. 一般不进行修改操作
    3. 只查表头元素 O(1)

栈

相当于数组操作的一个子集。后进先出Last In First Out（LIFO）

  • 时间复杂度

    • ArrayStack（数组栈）

      • push O(1) 均摊
      • pop O(1) 均摊
      • peek O(1)
      • getSize O(1)
      • isEmpty O(1)

    • LinkedListStack

      • push O(1)
      • pop O(1)
      • peek O(1)
      • getSize O(1)
      • isEmpty O(1)

    对比

    1. 复杂度相同，所以差异很小
    2. 但是当插入数据很多的时候，有可能链表栈效率会慢， 因为其new次数太多耗费时间，会去内存找新的可以开辟空间的地方

队列

先进先出 （FIFO）

数组队列

  • 时间复杂度
    • ArrayQueue（数组对列）
      • enqueue O(1) 均摊
      • dequeue O(n) 动态数组每次要往前移，重新赋值
      • front O(1)
      • getSize O(1)
      • isEmpty O(1)

循环队列

数组队列存在的问题：

删除队首元素时，O(n)，整体需要往前移一个位置，当n很大的时候，耗时

  • 时间复杂度
    • LoopQueue（循环对列）
      • enqueue O(1) 均摊
      • dequeue O(1) 均摊
      • front O(1)
      • getSize O(1)
      • isEmpty O(1)

链表队列

  • 思路历程

    1. LinkedList的头部增或删都是 O（1）级别的，而在尾部则需要O（n）级别搜索过去
    2. 因此可以用一个tail引用来记录尾部的地址，这样在尾部增操作也只需要O（1）级别操作。但在尾部删还是需要O（n），因为删除需要prev节点，还是需要遍历
    3. 综合考虑时间复杂度
      • 链表头head当做队首，作为删除元素
      • 尾tail当做队尾，来添加元素

  • 时间复杂度

    • LinkedListQueue
      • enqueue O(1)
      • dequeue O(1)
      • front O(1)
      • getSize O(1)
      • isEmpty O(1)
